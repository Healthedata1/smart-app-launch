SMART App Launch Framework

StructureDefinition-conformance-smartlaunch

Formats: Narrative, XML, JSON

The SMART on FHIR  Conformance defines extensions for automated discovery of OAuth2 endpoints and optional SMART launch features. These extensions are marked as “Must Support” whihc means servers SHALL include the extensions as part of their Conformance if the data is present or feature is supported and clients SHALL be capable of processing a CapabiltyStatement instances containing the data elements without generating an error an causing the application to fail.
